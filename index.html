CSS

  • A margin indicates how much space we want around the outside of an element.

  • Padding indicates how much space we want around the content inside an element.

  • The * symbol on the css file indicates that all elements will have that value in the curly brackets.

  • In css, separating the selectors *header , footer , ect.* with a comma will allow the declerations to affect all selectors implied.
    This will help in keeping lines of code to a minimum, AKA DRY code.
    (Do Not Repeat Yourself) code 😁

GIT

  • cd *The file you want*, will move you to that file, if in the directory.

  • cd .. moves back in git.

  • ls gives you a list of thigs in the directory.

  • pwd gives you the file path you are in.

  • git status checks what branch we are currently on

  • git checkout -b feature/ *discriptive name EXP. add-css,add-HTML,ect creates a new branch and switches to it.

  • git add -A adds the changed files in our working directory in GitHub.

  • git commit -m "" commits the changes to the repository.Keep them dicriptive and short.

  • git pull origin main pull the main branch to make sure that our current branch is
    up to date before opening a pull request in GitHub.

  • git push origin feature/ *File discription you made* will send the branch you have been working on to GitHub.
